# smart-home-network

Сетевая версия умного дома

Проект содержит:

- умную розетку, которая синхронно общается с удалённым устройством по TCP
- имитатор умной розетки с неблокирующим TCP-сервером и поддержкой нескольких клиентов
- умный термометр, который получает температуру UDP-пакетами в отдельном потоке
- имитатор термометра, который периодически отправляет псевдослучайные UDP-пакеты
- пример умного дома с сетевыми розетками и термометрами

## Запуск через скрипты

Собрать проект:

```bash
./build-app.sh
```

Запустить пример с автоматически поднятыми имитаторами:

```bash
./run-app.sh -- --start-simulators
```

Запустить пример и самому поднять имитаторы:

Терминал 1: запустить оба имитатора:

```bash
./run-app.sh --bin socket-simulator -- 127.0.0.1:4000 &
./run-app.sh --bin thermometer-simulator -- thermometer.conf
```

Первый процесс запускает TCP-розетку на `127.0.0.1:4000` в фоне.
Второй процесс запускает UDP-термометр, который отправляет температуру на `127.0.0.1:4001`.

Терминал 2: получить состояние устройств:

```bash
./run-app.sh
```

Пример без флага `--start-simulators` подключается к уже запущенной розетке и слушает термометр.
Если запустить только `socket-simulator`, пример завершится ошибкой:

```text
Error: failed to get device data: room 'lab': device 'network_thermometer': temperature has not been received yet
```

Это означает, что UDP-термометр не запущен или не отправляет данные на `127.0.0.1:4001`.

## Отдельные имитаторы

Имитатор TCP-розетки принимает адрес из аргументов:

```bash
./run-app.sh --bin socket-simulator -- 127.0.0.1:4000
```

Имитатор UDP-термометра читает адрес и период отправки из файла:

```text
target_addr = 127.0.0.1:4001
period_ms = 500
```

Запуск:

```bash
./run-app.sh --bin thermometer-simulator -- thermometer.conf
```
